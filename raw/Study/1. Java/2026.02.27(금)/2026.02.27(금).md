- 증감 연산자 (Java 교안 PDF p.23)(++ / --) (혼자서도 사용 가능)(ex : ++x ;)
- (But (x+y)++ 같이 수식에는 적용 X)
전위 : 연산 전에 증감
후위 : 연산 후에 증감

```java
// 변수 정의 및 대입
int a = 10, b = 20, c ;
c = ++a + b++ ; // 순서 : ++a, +, =, b++

// 출력
System.out.println("a : " + a); // 11
System.out.println("b : " + b); // 21
System.out.println("c : " + c); // 31

// 변수 재정의
c = a++ + --b ; // 순서 : --b, +, =, a++

// 출력
System.out.println("a : " + a); // 12
System.out.println("b : " + b); // 20
System.out.println("c : " + c); // 31

// 변수 재정의
a = 15;
b = 12;

c = --a + --b ; // 순서 : --a, --b, +, =

// 출력
System.out.println("a : " + a); // 14
System.out.println("b : " + b); // 11
System.out.println("c : " + c); // 25


// 변수 재정의
a = 10;
b = 20;
++a ; // 11 // 단독으로도 사용 가능!
b-- ; // 19 // 단독으로도 사용 가능!

c = ++a + b-- ; // 순서 : ++a, +, =, b--

// 출력
System.out.println("a : " + a); // 12
System.out.println("b : " + b); // 18
System.out.println("c : " + c); // 31
```

이 의미는 👉 서로 공통 조상이 없는 두 저장소를 병합하려고 할 때 발생합니다.

보통 이런 상황입니다:

GitHub(예: GitHub)에서 먼저 README 생성

로컬에서 따로 git init 후 작업

이후 git pull 시도

→ 서로 완전히 다른 시작점이라 Git이 병합을 거부합니다.

✅ 해결 방법 1 (가장 일반적)

강제로 병합 허용:

git pull origin master --allow-unrelated-histories

👉 한 번만 실행하면 됩니다.

충돌이 나면 수동으로 해결 후:

git add .
git commit
✅ 해결 방법 2 (로컬 내용 유지하고 싶을 때)

원격 내용을 무시하고 강제로 덮어쓰기:

git push -f origin master

⚠ 주의: 원격 저장소 내용이 사라집니다.

✅ 해결 방법 3 (원격 기준으로 새로 시작)

로컬 내용이 필요 없으면:

git fetch origin
git reset --hard origin/master
🔎 왜 이런 일이 생겼을까?

현재 로그를 보면:

\* branch master -> FETCH_HEAD
= [up to date] master -> origin/master

원격은 업데이트 없는데 병합 단계에서 거부됨 →
👉 과거에 git init 방식이 서로 다르게 시작된 상태.

💡 가장 안전한 방법

대부분은 이것만 하면 해결됩니다:

git pull origin master --allow-unrelated-histories

혹시 지금 상황이:

git init 

git add .

git commit -m "second java"

git push -f origin master
