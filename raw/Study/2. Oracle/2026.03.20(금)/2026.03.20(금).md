### # 이상 현상
데이터 집합에서 예상하치 못한, 비정상적인 패턴이나 동작을 나타내는 현상
ex) 데이터의 오류, 이상치, 누락, 부정확성 등
이상치 : 상식선에서 정상적인 범위를 벗어난 경우

1) 삽입 이상 (Insert) : 데이터 삽입시 필요 없는 속성까지 입력해야 하는 현상을 말합니다.
ex) '통계학'이라는 신설 학과를 등록하는 경우, 학생 정보와 과목에 대한 정보를 입력해야 하는 문제가 있습니다.

2) 갱신 이상 (Update) : 데이터 갱신시 필요하지 않는 데이터까지 갱신이 되는 현상을 말합니다.
일반적으로 동일한 데이터에 대하여 2건 이상의 업데이트가 일어나는 경우에 많이 발생합니다.
ex) '신동진' 학생의 주민 등록 번호가 변경이 되었는데, 데이터 중복으로 인하여 다른 행이 갱신되지 않는 문제가 있습니다.
일반적으로 갱신 이상은 동일한 데이터에 대하여 2건 이상의 업데이트가 일어나는 경우에 많이 발생합니다

3) 삭제 이상 (Delete) : 데이터 삭제시 보존이 되어야 하는 데이터까지 삭제가 되는 현상을 말합니다.
ex) 퇴학된 학생 '이가희'가 삭제될 때, 대학 학과 정보(의대 정보)가 동시에 삭제가 되는 문제가 있습니다.

	"OID" NUMBER(38,0) NOT NULL,

	ID VARCHAR2(30) NULL,
	CONSTRAINT ORDERS_OID_PK PRIMARY KEY ("OID")

CREATE TABLE BLUESKY.PRODUCTS (
	PNUM VARCHAR2(30) NOT NULL,
	DESCRIPTION VARCHAR2(255) NULL,
	CONSTRAINT PRODUCTS_PNUM_PK PRIMARY KEY (PNUM)

	"OID" NUMBER(38,0) NULL,
