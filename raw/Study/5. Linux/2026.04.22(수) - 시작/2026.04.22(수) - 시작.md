#### \# 리눅스 Prompt 구조 (Linux 이론.pdf(p.5))
사용자가 터미널에서 명령을 입력할 수 있도록 표시하는 텍스트
root : 로그인한 사용자 계정명
localhost : 리눅스 시스템의 호스트명(/etc/sysconifg/network)
~ : 현재 작업 디렉토리 위치(~ 기호는 Home Directory를 의미)
#### \# : 관리자용 prompt
$ : 일반 사용자 prompt

ex)
root@myserver:~#
root (접속 중인 계정 정보)
myserver (host이름 - 컴퓨터 이름)
~ (현재 접속 위치) (~은 home directory라고 부름)
#### \# (관리자용)

ugcadman@myserver:~$
ugcadman (접속 중인 계정 정보)
myserver (host이름 - 컴퓨터 이름)
~ (현재 접속 위치)
$ (일반 사용자용)

\* pwd를 입력하면 현재 디렉토리의 경로가 나옴 (MobaXterm에서)
ex)
broadcast@Broadcast:~$ pwd
/home/broadcast

주소 전체를 모두 표현하는 방식

서버 관리나 스크립트 작성 시에는 절대 경로를 많이 사용합니다.
위치가 바뀌어도 항상 동일한 결과를 보장합니다.

ex) /home/broadcast/kbs/1night_2days/hello.txt


#### \# 상대 경로(Relative Path) (Linux 이론.pdf(p.8))
지금 내가 있는 위치를 기준으로 어디로 갈지
현재 작업 중인 디렉토리 기준
/ 로 시작하지 않음
현재 위치에 따라 결과가 달라짐

- 주요 기호
. 현재 디렉토리
.. 상위 디렉토리
./ 현재 위치 기준
../ 한 단계 위로 이동
~/ Home 디렉토리를 의미함

ex)
(현재 위치: /home/broadcast)
kbs/1night_2days/hello.txt
