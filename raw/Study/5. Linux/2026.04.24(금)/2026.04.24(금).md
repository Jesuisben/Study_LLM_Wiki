1) - : 파일을 뜻함
2) rw- (앞에꺼)(Read Write) : Owner permissions (소유자꺼) - 해당 파일에 대해 읽기 / 쓰기 / 실행 가능
permissions은 Access Rights - 접근 권한
r(Read) - cat을 할 수 있다.
w(Write) - echo 할 수 있다.
\* x는 execute (실행) - .sh(shell)을 실행할 수 있다.
\* 리눅스는 멀티 개발 환경이여서 이런 개념이 있음

3) rw- (뒤에꺼)(Read Write) : Group permissions (소유자를 포함한 더 큰 개념)
4) r-- : Others permissions (그외 나머지)
5) broadcast (앞에꺼) :
6) broadcast (뒤에꺼) :
7) 80 : 파일사이즈 (bytes 단위)
8) Apr 23 03:58 : 마지막으로 수정된 시간
9) java.txt : 파일 이름

#### \# 요구사항에 부합하는 사용자 생성 (Linux 실습(MobaXterm, VirtualBox, 실습).pdf(P.134))
(rainbow7 사용자 계정 생성하기)

- 요구 사항
사용자 : rainbow7
홈 디렉토리 : /home/rainbow7
코멘트 : shy_guy
계정 사용 종료일 : 2025-12-25
UID : 7500
Shell : /bin/bash
비밀 번호 : test1234

- 해보기
1) -p 옵션은 평문 비밀 번호가 아니라 암호화(해시)된 비밀 번호를 직접 넣을 때 사용하는 옵션입니다.
useradd -m -c shy_guy -e 2025-12-25 -d /home/rainbow7 -u 7500 -s /bin/bash -p $(openssl passwd -6 test1234) rainbow7
(풀이)
-m : 홈 디렉토리를 자동으로 생성
-c : (Comment) 설명(주석) 추가
-e : (Expire Date) 계정 만료일 지정
-d : (Directory) 홈 디렉토리의 경로를 지정
-u : (UID) UID를 지정
-s : (Shell) 기본 쉘을 지정
-p : (Password) 암호화된 비밀번호를 직접 전달함
-> -p 옵션은 평문(test1234)을 그래도 넣으면 안됨. 반드시 해시화(암호화)된 값을 넣어야 함
-> $(...) 구문은 괄호 안의 명령어를 먼저 실행한 결과값을 가져옴
-> penssl passwd -6 test1234 : test1234라는 비밀번호를 SHA-512 방식(-6)으로 암호화하라는 뜻

tail /etc/passwd | grep rainbow7
-> rainbow7:x:7500:7500:shy_guy:/home/rainbow7:/bin/bash

tail /etc/group | grep rainbow7
-> rainbow7:x:7000:


4) 시스템 계정은 보통 UID가 0~999이고, 일반 사용자 목록은 UID가 1000 이상
필드 구분자 :으로 일반 사용자 보기
awk -F: '$3 >= 1000 && $3 < 65534 { print $1 }' /etc/passwd

\* 꼭 쌍따옴표가 아니고 그냥 따옴표를 사용해야 함 *
(풀이)

-F: : 필드구분자를 :(콜론)으로 설정
$3 : 3번째 열 (UID)
$3 >= 1000 && $3 < 65534 : 3번째 열 (UID)이 1000 이상이고 65534 미만인 것들의
{ print $1 } : 첫번째 열 (파일 이름)을 출력하라
/etc/passwd : 해당 경로에 있는 파일들 중에서

#### \# chmod 명령어 - 기호 모드 (Linux 이론.pdf(P.121))
문법 :

chmod o=r file.txt : 기타 사용자에게 읽기 권한만 설정
chmod g+rx test test : 파일의 그룹에 읽기와 실행 권한을 허가(symbolic method)
