1) Temurin은 Eclipse 재단의 OpenJDK(Java 개발 키트) 무료 배포판으로 JDK가 포함(JRE + 컴파일 도구)되어 있음


2) Maven의 JAR 파일 위치 (target/*.jar) : Docker 이미지 빌드 시점에 전달 받을 매개 변수로 target/폴더 아래 모든 jar 파일을 의미
ARG JAR_FILE=target/*.jar

3) 호스트 PC의 JAR 파일을 컨테이너 내부로 docker_spring_mysql.jar라는 이름으로 복사하기
기호 ${ }는 "변수 치환(variable expansion)" 문법임

```dockerfile
COPY ${JAR_FILE} docker_spring_mysql.jar
```

5) 스프링 부트 애플리케이션 실행
컨테이너가 시작이 될 때 java -jar /docker_spring_mysql.jar라는 명령어를 자동 실행함을 의미

```dockerfile
ENTRYPOINT ["java", "-jar", "/docker_spring_mysql.jar"]
```

상담사를 docker container로 이해하면 쉬움 (웹서비스를 해주는 서버)
처리해야할 것들이 많은데 그것을 처리하는 container 2개중에서 하나만 사용하면 업무 부하가 심해지고

3) Reverse Proxy용 nginx 설정 파일 생성 (프록시 서버를 nginx로 만들 예정)
3-1) nginx 설정 파일 생성 및 편집기로 들어가기

```bash
sudo vi ~/nginx.conf
```

3-2) 아래의 코드 복붙하고 저장하고 나오기
#### \# nginx.conf 파일 시작
events {}

http {
#### \# upstream : 서버 그룹 (로드밸런싱)
#### \# backend_apache라고 그룹 이름 설정
    upstream backend_apache {
        server apache01:80;
        server apache02:80;
        server apache03:80;
    }

    upstream backend_nginx {
        server nginx04:80;
        server nginx05:80;
        server nginx06:80;
    }

    server { # 외부 요청 받는 곳
#### \# 기본 포트를 80으로 지정
        listen 80;

#### \# Apache로 보내기
#### \# 만약 /apache라고 치면 해당 경로(그룹)로 이동하게 하기
        location /apache/ {
            proxy_pass http://backend_apache/;
        }

#### \# Nginx로 보내기
        location /nginx/ {
            proxy_pass http://backend_nginx/;
        }

#### \# 기본 (로드밸런싱 테스트)
        location / {
            proxy_pass http://backend_apache/;
        }
    }
}
#### \# nginx.conf 파일 끝


4) Reverse Proxy nginx 컨테이너 실행
- 호스트에 있는 nginx 설정 파일을 컨테이너 안으로 “연결(마운트)”해서 사용하겠다는 의미
```bash
docker container run -d --name reverse-proxy --network proxy-net -p 80:80 -v $(pwd)/nginx.conf:/etc/nginx/nginx.conf nginx
```
